# BRSCCP Log Parser

`parse_brsccp.py` — это инструмент командной строки для преобразования текстовых лог-файлов BRSCCP в структурированный формат (JSON или CSV). Скрипт разработан с упором на надежность, обработку аномальных завершений и сохранение временных меток событий.

## Основные возможности

- **Наследование таймстампов**: Автоматическое проставление времени для строк лога, которые его не содержат.
- **Умный парсинг параметров**: Использование `ast.literal_eval` для точного приведения типов входных параметров (списки, словари, числа) и корректная обработка многострочных записей.
- **Отказоустойчивость**: Каждая стадия парсинга изолирована; ошибки в одном блоке не прерывают обработку всего файла, а записываются в `parse_warnings`.
- **Поддержка ABNORMAL TERMINATION**: Скрипт корректно обрабатывает незавершенные логи, возвращая доступную диагностику даже при отсутствии блока итогов.
- **Гибкий вывод**: Поддержка форматов JSON (для полной структуры) и CSV (для агрегации ключевых метрик).

## Установка

Скрипт использует только стандартную библиотеку Python 3.9+ и не требует установки дополнительных пакетов.

```bash
# Клонируйте репозиторий или скачайте файл
python3 parse_brsccp.py --help
```

## Использование

### Базовая команда
```bash
python3 parse_brsccp.py path/to/logfile.log
```

### Параметры командной строки
- `log`: Путь к лог-файлу (обязательный).
- `--format {json,csv}`: Формат вывода (по умолчанию `json`).
- `--out FILE`: Сохранить результат в указанный файл.
- `--strict`: Завершать работу с ненулевым кодом, если обнаружены предупреждения парсера или статус лога не `NORMAL`.
- `--self-check`: Запустить встроенный тест на эталонном логе.

### Примеры
```bash
# Сохранить результаты в CSV для таблицы
python3 parse_brsccp.py my_run.log --format csv --out summary.csv

# Использование в пайплайнах с проверкой ошибок
python3 parse_brsccp.py my_run.log --strict
```

## Модель данных

### JSON Структура
Объект `BrsccpLog` содержит следующие блоки:
- `version`, `build_date`, `start_time`: Информация о версии программы и времени запуска.
- `input_params`: Словарь всех параметров из секции "Input parameters".
- `results`: (Опционально) Блок с энергиями `EnergyState` для двух состояний, `ddG_kcal_mol` и `activity_ratio`.
- `diagnostics`: Траектории оптимизации, список альтернативных структур, аномальные сдвиги атомов (`atom_shifts`) и предупреждения PDB.
- `termination`: Статус завершения (`NORMAL`, `ABNORMAL` или `UNKNOWN`).
- `parse_warnings`: Список проблем, возникших непосредственно при разборе файла.

### CSV Поля
При выборе CSV формата выводится одна строка со следующими данными:
`model_name, transformation, ddG, activity_ratio, termination, shift_count`
